1.3 - Getting Started & Installation

There is only one js file that is required to include. 10kb

It's really easy to implement and lightweight!

For basic implementation of our plugin,
Firstly, you need to add the "Pure.Uploader.js" file and the Pure.Uploader.Basic.css to the <head> section 

						
<script type="text/javascript" src="Pure.Uploader.js"></script>
<link rel="stylesheet" type="text/css" href="Pure.Uploader.Basic.css" />
Secondly, you need the HTML elements that our plugin needs,
careful about the IDs, you are going to need these to identify these elements to our plugin 
						
	<form id="dropper">
            Pick some files 
            <input type="file" name="fileInput" id="fileInput" multiple />
            or drop some here ⇓
            <div id="dropPlace"></div>
            <button type="submit">Start Upload</button>
            <div id="imageHolder"></div>
        </form>
Lastly, you need to initialize our plugin with the settings you need 
						
<script type="text/javascript">
        var d = new PureUploader(
        {
            drop: true, // to support drag&drop
            form: document.getElementById("dropper"), // the form element's id
            fileInput: document.getElementById("fileInput"), // the file type input's id
            imageHolder: document.getElementById("imageHolder"), // the div's id for listing of uploading files
            dropPlace: document.getElementById("dropPlace"), // the div's id that images gonna drop onto
            dragHoverClass: "drop_hover", // when the state of hover of the dropPlace div this class would approved
	// Thumbnail resolutions will being used in listing of uploads and thumbnails that would create if the setting is on
                image: {
                    thumb: true, //active thumbnail
                    thumb_width: 200, //thumbnail width
                    thumb_height: 200, //thumbnail height
                    resize_width: 0, //image resize width
                    resize_height: 0, //image resize height
                    keep_aspect_ratio: true //image resize aspect ratio
                },
	// File filter that which files do you want to uploaded with this plugin enter these like this png|jpg|rar|doc etc...
	// If this is empty (""), this means you allow all the file types
            file_filter: "",
            
	// Ajax settings
	// url : the url of the ajax request would send, clearAfterUpload : a boolean setting that for if you want to clear items after the upload process
            ajax: {
                url: '/Home/Receive/',
                clearAfterUpload: true
            },
			
	// Watermark settings, if you don't want to watermark images leave it, if you do see watermark settings	see downside	
            watermark: {
                watermark: false
            },
        //Icon Properties for files
            icon: { 
                icon : true, //active icons
                path: "/images/icons", //icons path
                _default: "/images/icons/_blank.png", //default icon for not supported files types
                width: 128, //Icon width
                height: 128 //Icon height
            },
        //Auto upload
	        auto_upload:false,
	// HTML5Error event to handle when the browser doesn't support HTML5 like File API and Drag&Drop features that plugin needs let the user know by this
	// and will use simple uploading system instead of HTML5 one
            html5Error:
                function (uploader) {
                    uploader.settings.imageHolder.style.display = "none";
                    uploader.settings.dropPlace.style.display = "none";
                    var error = document.createElement("p");
                    error.className = "text-center";
                    error.appendChild(document.createTextNode("Your browser doesn't support HTML5, we can offer you a new browser from here ! click!"));
                    uploader.settings.form.appendChild(error);
                },
				
	// Progress event: this one retrieves the progress of each item that is being uploaded to show the progress
            progress:
                         function (data) {
                             var template = document.getElementById(data.template); // to access each individual item, required.
                             if (template) {
                                 var progress = document.getElementById("progress_" + data.template); // to access and manipulate the progress of each individual, required.
				//from this state you can customize below to fit your needs
                                 if (progress) {
                                     progress.innerText = "%" + data.percent;
                                 }
                                 else {
                                     var div = document.createElement("div");
                                     div.id = "progress_" + data.template;
                                     div.innerText = "%" + data.percent;
                                     template.appendChild(div);
                                 }
                             }
                         }
        });
    </script>
Done for basic version

2.2 - Functions

Function Description Parametres
html5Error {function} When browser doesn't support HTML5, this will be fired none
success {function} When a file uploaded to server succesfully,this will be fired XMLHttpRequestProgressEvent
progress {function} While a file is being uploaded it will be return percent value and div id { percent: "100", template: "templateid from imageholder" }
error {function} When error occured, this will be fired. uploaderObject - ErrorMessage - ErrorObject
beforeSend {function} When before send files, this will be fired.
Important ! Its child of ajax properties like ajax : { beforeSend : function(xhr) { //Do your thing } } 		xhr - XMLHttpRequest
Direct Functions new important

This functions are calling with uploader object.

For example; You did a instance of PureUploader

                                var myUploader = new PureUploader({.....});
                                myUploader.start('temp_123441411', function(){ console.log("My job is done.") });

Note : If a function has foo parameters, its mean you can give it a function for after the job completed.

Function Description Parametres
start {function} It will start the upload of the item. id - foo
pause {function} It will pause the upload of the item. id - foo
remove {function} It will remove the upload and abort the XHR object. id - foo
find {function} Find the object of item. id
isworking {function} If there is a uploading item it will be return true.
Important ! Its child of ajax properties like ajax : { beforeSend : function(xhr) { //Do your thing } } 		none

2.3 - Template new important

This attribute is making more customize the uploader. It is working simple, but you need to be careful on usage. You should don't forget some keys.

Default value

                        
    template:
        '<div class="form-group text-center">' +
            '<a href="javascript:void(0)" class="btn btn-danger close" onclick="{uploader}.remove(\'{id}\')">x</a>' +
            '{image}' +
        '</div>' +
        '<div class="form-group text-center">' +
            '<div class="form-group col-md-6">' +
                '<a href="javascript:void(0)" class="btn btn-info" onclick="{uploader}.start(\'{id}\')">Start</a>' +
            '</div>' +
            '<div class="form-group col-md-6">' +
                '<a href="javascript:void(0)" class="btn btn-info" onclick="{uploader}.pause(\'{id}\')">Pause</a>' +
            '</div>' +
            '<p>{file.name} - {file.size}</p>' +
        '</div>',

As you see at the above, there are keys like {uploader},{image},{id},{file.name},{file.size}

Key Description
{uploader} It will return the uploader name which is you will give it on settings.
{image} It will be image position on the html side.
{id} It will return the id of item.
{file.name} Just file name.
{file.size} Just file size.

3.1 - Overview

For Html5 :

You must handle your html5 upload with base64 stream type, which like we've done via ASP.NET

After handling base64, you would return JSON result whatever your want

For Oldschool :

You must handle like regular upload. It's based on your server language for example in ASP.NET it's HttpPostedFileBase

3.3 - Asp.NET

We have developed our dll for .net applications, it will come with the plugin you can use

There are 2 functions

TFile:
  1. UploadAjax(Stream = base64, string = fileName, Bool = thumbnail)
  2. UploadAjax(HttpPostedFileBase = file, Bool = thumbnail)
Asp.NET MVC 4 Example
Ajax: 
        [HttpPost]
        public ActionResult Receive()
        {
            bool thumb = Convert.ToBoolean(Request.Headers["Uploader-Thumb"]);
            int thumb_width = Convert.ToInt32(Request.Headers["Uploader-ThumbHeight"]);
            int thumb_height = Convert.ToInt32(Request.Headers["Uploader-ThumbWidth"]);
            string name = Request.Headers["Uploader-Name"];
            
            TFile file = new TFile(Server.MapPath("~/uploads/images"), Server.MapPath("~/uploads/thumbnails"), thumb_width, thumb_height);
            if (file.UploadAjax(Request.InputStream, name, thumb) == -1) //fail
            {
                return Json(false);
            }
            else //success
            {
                return Json(true);
            }
        }
